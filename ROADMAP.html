Что работает

Открой localhost:3000 и пользуйся

01

Счета · карты

Дебетовые, накопительные, валютные, брокерские. Свой цвет и плашка для каждого.

02

Новая операция

Расход/доход в одно действие. Категория, дата, мерчант. Баланс пересчитывается сразу.

03

Переводы между счетами

Атомарно. История показывает обе стороны через transfer_group_id.

04

Аналитика

Категории за 30 дней с цветными барами и поток денег по месяцам.

05

Инвестиции · MOEX

Наблюдение, портфель, покупка/продажа. Котировки SBER, YNDX, LKOH, GAZP, VTBR — с биржи, не моки.

06

Кредиты · вклады

Калькулятор аннуитетных платежей (PMT). Подаёшь заявку → видишь ежемесячный платёж.

07

История

Все операции, сгруппированные по дням. Цветные плашки категорий.

08

Бэкап

Профиль → «Скачать бэкап» (.json). Импорт работает тоже. Перенесёшь данные на другой комп.

Как запустить

3 минуты

1

Локально (для разработки)

Открой терминал в папке проекта:

cd web
npm install
npm run dev

# открой http://localhost:3000
2

На GitHub Pages (бесплатно навсегда)

Создай репозиторий на GitHub, например T-Finance. Запушь проект:

git init
git add -A
git commit -m "init"
git branch -M main
git remote add origin https://github.com/<твой-username>/<имя-репо>.git
git push -u origin main
3

Включи Pages

В репозитории на GitHub:

Settings → Pages → Source: GitHub Actions

Всё. Workflow в .github/workflows/deploy.yml запустится сам и через ~2 минуты сайт будет на
https://<username>.github.io/<имя-репо>/

Каждый последующий git push = автоматический деплой.

Что важно знать

Без сюрпризов потом

бэкап

Данные живут только в твоём браузере. Сменишь компьютер / почистишь кэш / поедешь в режим инкогнито — данные не подтянутся. Скачивай бэкап раз в неделю (Профиль → Скачать бэкап). Это .json файл, его можно загрузить обратно одной кнопкой.

телефон ≠ комп

На телефоне и на компе будут разные базы. Если захочется синхронизации между устройствами — переключим на серверный режим (бэкенд уже написан, лежит в backend/). Это работа на один день.
