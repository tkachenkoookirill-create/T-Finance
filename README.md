# T-Finance

Личный финансовый трекер. Все данные хранятся в твоём браузере — никаких серверов, паролей, аккаунтов.

**Открой `T-Finance.html`** чтобы посмотреть дизайн.
**Открой `ROADMAP.html`** чтобы увидеть карту проекта.

## Запуск локально

```bash
cd web
npm install
npm run dev
# открой http://localhost:3000
```

При первом открытии создадутся демо-счета и пара операций — чтобы было что посмотреть. Можешь их удалить и завести свои.

## Деплой на GitHub Pages (бесплатно, навсегда)

1. Создай новый репозиторий на GitHub, например `T-Finance` или `my-finance`.
2. Запушь весь этот проект:
   ```bash
   git init
   git add -A
   git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/<твой-username>/<твой-репо>.git
   git push -u origin main
   ```
3. На GitHub в репозитории зайди в **Settings → Pages → Source → GitHub Actions**.
4. Готово. После каждого `git push` сайт автоматически обновляется на
   `https://<твой-username>.github.io/<твой-репо>/`

Workflow уже лежит в `.github/workflows/deploy.yml` — ничего настраивать руками не надо.

## Что умеет

- **Счета** — несколько разных (наличные / накопительный / валютный / брокерский), цвета и плашки
- **Операции** — добавление вручную, расход/доход, категория, дата
- **Переводы** — между своими счетами, атомарно (история показывает обе стороны)
- **История** — все операции, сгруппированные по дням
- **Аналитика** — траты по категориям + поток денег по месяцам
- **Инвестиции** — наблюдение и портфель, котировки тянутся в реальном времени с **Московской биржи** (`iss.moex.com`, бесплатно, без ключа)
- **Кредиты/вклады** — калькулятор аннуитетных платежей по продуктам
- **Бэкап** — Профиль → Скачать бэкап (.json) / Загрузить из файла

## Где хранятся данные

В `localStorage` твоего браузера. Это значит:
- ✅ Никто кроме тебя их не видит
- ✅ Работает офлайн (после первой загрузки)
- ⚠️ Данные привязаны к одному браузеру — **обязательно делай бэкап в Профиле**, прежде чем чистить кэш или менять устройство
- ⚠️ На телефоне и компе будут разные данные (если нужна синхронизация — переходим на серверный вариант, см. ниже)

## Структура

```
T-Finance/
├── T-Finance.html              # Дизайн-исходник (все экраны в одном файле)
├── ROADMAP.html                # Визуальная карта проекта
├── web/                        # ← основной код (Next.js + Tailwind)
│   └── src/
│       ├── app/
│       │   ├── globals.css     # дизайн-токены (oklch)
│       │   └── (app)/          # экраны: dashboard / accounts / add / ...
│       ├── components/         # Card · Pill · Button · Sparkline · Sidebar
│       └── lib/
│           ├── api.ts          # client-side store + MOEX интеграция
│           ├── money.ts        # форматтеры (минорные единицы)
│           └── i18n.ts
├── mobile/                     # Expo (на будущее, не используется на GitHub Pages)
├── backend/                    # FastAPI (на будущее, для синхронизации между устройствами)
└── .github/workflows/deploy.yml  # авто-деплой на Pages
```

## На будущее

Если когда-нибудь захочется:
- **синхронизация между телефоном и компом** → подключаем `backend/` (он уже написан) и выкладываем на Render/Railway
